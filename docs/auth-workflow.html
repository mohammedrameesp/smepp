SME++ Authentication Workflow

Complete guide to user authentication, signup, and onboarding flows

🔎 Overview

SME++ uses a unified authentication flow where all users (OAuth or Email/Password) follow the same path: Create Account → Onboarding → Create Organization → Dashboard. The middleware handles smart redirects based on authentication and organization status.

🔗 Route Structure

Route Type Description Layout
/ Public Marketing landing page (unauthenticated) or redirect to /admin (authenticated) Marketing
/pricing Public Pricing plans page Marketing
/login Public Sign in with Google, Microsoft, or Email/Password Auth (standalone)
/signup Public Create new account Auth (standalone)
/onboarding Auth Required Create organization (first-time setup) Auth (standalone)
/admin/* Org Required Admin dashboard and all admin features App Shell
/employee/* Org Required Employee portal App Shell

🚀 Smart Root Redirect

The middleware intercepts requests to / and redirects based on the user's state:

When user visits / (root):

  • Not authenticated → Show marketing landing page
  • Authenticated + Has Organization → Redirect to /admin
  • Authenticated + No Organization → Redirect to /onboarding

👤 User Scenarios

1 New User (Email Signup)

  • Visits /signup
  • Fills name, email, password
  • Account created in database
  • Auto signed in
  • Redirected to /onboarding
  • Creates organization
  • Redirected to /admin

2 New User (Google/Microsoft)

  • Clicks "Continue with Google"
  • Authenticates with OAuth provider
  • Account created automatically
  • Callback redirects to /
  • Middleware redirects to /onboarding
  • Creates organization
  • Redirected to /admin

3 Returning User (Has Org)

  • Visits /login
  • Signs in (any method)
  • Callback/redirect to /
  • Middleware detects org exists
  • Redirected directly to /admin

4 Returning User (No Org)

  • Previously signed up but didn't complete onboarding
  • Signs in again
  • Callback/redirect to /
  • Middleware detects no org
  • Redirected to /onboarding

🔒 Authentication Providers

📧

Email/Password

Traditional signup with email verification. User creates password, account stored in database with bcrypt hash.

Google OAuth

Sign in with Google account. User automatically created on first login. Profile synced from Google.

Microsoft Azure AD

Enterprise SSO with Microsoft 365. Ideal for businesses already using Microsoft ecosystem.
